# Next.js (App Router) 実装の学習まとめ

`books/page.tsx` の実装を通して学んだ、Next.jsのモダンなWeb開発における5つの重要なポイントです。今後の実装の参考にしてください。

---

## 1. Server Components (RSC) の基本
Next.js（App Router）では、デフォルトでコンポーネントが**サーバー側で実行**（React Server Components）されます。

```tsx
export default async function BooksPage() {
  const books = await getBooks();
  // ...
}
```
- **特徴:** コンポーネント自体を `async` 関数にできます。直接 `await` を使ってデータを取得し、データが揃ってからHTMLを生成してブラウザに返します。
- **メリット:** クライアント（ブラウザ）側に余計なJavaScriptが送られないため、ページの読み込みが速く、SEO（検索エンジン最適化）にも有利です。

## 2. 最新データの取得 (Data Fetching) とキャッシュ制御
APIからデータを取得する際、Next.jsの拡張された `fetch` 関数を使用しました。

```tsx
async function getBooks(): Promise<Book[]> {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL || "http://localhost:3001";
  const res = await fetch(`${apiUrl}/books`, { cache: "no-store" });
  return res.json();
}
```
- **動的レンダリング:** `{ cache: "no-store" }` を指定することで、「キャッシュせず、アクセスするたびに常に最新の情報をAPIから取得する」設定になります。
- **環境変数:** `process.env.NEXT_PUBLIC_API_URL` のように `NEXT_PUBLIC_` をつけると、ブラウザ側からもアクセスできる環境変数になります。

## 3. Server Actions によるフォーム送信
JavaScript（`onSubmit` や `useState`）を書かずに、フォームデータをサーバーに送信する仕組みです。

```tsx
import { createBookAction } from "./actions";

<form action={createBookAction}>
  {/* 入力フィールド */}
  <button type="submit">登録する</button>
</form>
```
- **特徴:** `<form>` の `action` 属性に、サーバー側で実行される関数（Server Action）を直接渡せます。
- **メリット:** クライアント側でAPIを叩くコードが不要になり、JavaScriptが無効な環境でもフォームが動作する堅牢な手法です。

## 4. 画像の最適化 (`<Image>` コンポーネント)
標準の `<img>` タグではなく、Next.jsが提供する `next/image` を使用しました。

- **最適化と遅延読み込み:** 画像を自動的に軽量なフォーマットに変換し、画面に表示されるタイミングで読み込みます（Lazy Load）。
- **CLS（レイアウトシフト）の防止:** `width` と `height` を指定し、画像が読み込まれる前にあらかじめスペースを確保することで、画面のズレを防ぎます。

## 5. Tailwind CSS によるモダンなスタイリング
CSSファイルを別で作成せず、クラス名だけでリッチなUIを構築しました。

- **レスポンシブデザイン:** `sm:` や `lg:` のプレフィックスで、画面幅に応じたレイアウトの切り替え（例: スマホなら1列、PCなら3列）を直感的に実装できます。
- **インタラクション:** `group` と `group-hover` を組み合わせることで、「カードにホバーした時に中の画像やテキストを変化させる」高度なアニメーションを実現できます。
